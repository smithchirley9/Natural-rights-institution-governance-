SECTION 2 — SECURITY OWNERSHIP MODEL (March 19)

Invariant:
No single person can change bytes, semantics, enforcement, and deploy.

Roles:

Protocol Owner

- Owns semantic meaning
- Cannot approve runtime or deploy

Implementation Owner

- Writes code
- Cannot approve policy

Oracle / Fixture Owner

- Defines expected behavior
- Must cross-review

CI / Runtime Owner

- Controls execution environment
- Cannot change semantics

Release Authority

- Only role allowed to deploy
- Requires approvals

Rule:
Pressure never overrides approval chain.
